# =============================================================
# CONSTRUCTED HARD INSTANCES — Bellman-Ford-Moore
# These cases are designed to stress-test the algorithm by
# forcing worst-case relaxation behavior, near-negative cycles,
# long chain dependencies, and large dense graphs.
# =============================================================

# ─────────────────────────────────────────────────────────────
# Hard Case 1 — Long relaxation chain (Adjacency List)
# 8 vertices, 7 edges, source = 0
# Purpose: Forces maximum number of relaxation passes.
#   Edges are ordered WORST-FIRST (reverse of optimal order),
#   so no edge can be finalized until the previous vertex is
#   settled. Bellman-Ford needs V-1 = 7 full passes to converge
#   instead of converging early. This is the classic worst-case
#   edge ordering that defeats early-termination optimizations.
# Expected: dist = [0, -1, -2, -3, -4, -5, -6, -7]
# ─────────────────────────────────────────────────────────────
LIST
8
7
6 7 -1
5 6 -1
4 5 -1
3 4 -1
2 3 -1
1 2 -1
0 1 -1
0

# ─────────────────────────────────────────────────────────────
# Hard Case 2 — Near-negative cycle (Adjacency List)
# 5 vertices, 6 edges, source = 0
# Purpose: A cycle whose total weight is exactly 0 (not negative).
#   Cycle: 1 → 2 → 3 → 1  with weights 3 + (-2) + (-1) = 0
#   Tests that the algorithm does NOT falsely flag a zero-weight
#   cycle as a negative cycle — a common off-by-one bug.
#   Also tests that shortest paths through the cycle are still
#   computed correctly.
# Expected: no negative cycle detected, valid distances returned
# ─────────────────────────────────────────────────────────────
LIST
5
6
0 1 5
1 2 3
2 3 -2
3 1 -1
0 3 2
3 4 4
0

# ─────────────────────────────────────────────────────────────
# Hard Case 4 — Multiple negative cycles, only some reachable
# (Adjacency List)
# 7 vertices, 10 edges, source = 0
# Purpose: Two negative cycles exist in the graph, but only one
#   is reachable from source 0. Tests that the algorithm flags
#   the negative cycle correctly while handling the unreachable
#   cycle (vertices 5-6) gracefully without a false positive.
#   This is a subtle correctness trap — naive implementations
#   may flag both or neither.
# Reachable negative cycle: 1 → 2 → 3 → 1  (1 + 1 + (-3) = -1)
# Unreachable negative cycle: 5 → 6 → 5    (-2 + (-2) = -4)
# Expected: negative cycle detected (reachable one triggers it)
# ─────────────────────────────────────────────────────────────
